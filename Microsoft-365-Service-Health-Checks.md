# Microsoft 365 Service Health Checks

## Purpose

Service health checks help identify whether a reported issue is caused by a wider Microsoft service incident or by a local user, device, or configuration problem.

## Basic Process

1. Confirm the affected Microsoft 365 service, such as Outlook, Teams, OneDrive, or Exchange Online.
2. Ask the user what error message they are seeing and when the issue started.
3. Check whether multiple users are reporting the same issue.
4. Review the Microsoft 365 Admin Center service health area.
5. Check active incidents, advisories, and recent service history.
6. Compare the reported symptoms with Microsoft's incident details.
7. If there is no wider outage, continue with local troubleshooting.
8. Record findings and keep affected users updated.

## Information to Record

- Affected service
- Start time
- Number of affected users
- Error messages
- Service health findings
- Troubleshooting already completed
- Resolution or escalation details

## Key Lesson

Checking service health early can prevent unnecessary troubleshooting and helps the support team communicate accurate updates to users.
